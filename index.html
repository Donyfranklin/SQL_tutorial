Introduction

🗄️ Welcome to SQL

SQL (Structured Query Language) is how you talk to a database. Think of a database as a very organized Excel file — and SQL is the language you use to read, add, edit or delete data in it.

🗃️

Database

A collection of organized tables. Like an Excel workbook.

📋

Table

A grid of rows and columns. Like one Excel sheet.

📝

Row (Record)

One entry of data. Like one student's information.

🏷️

Column (Field)

A category of data. Like "Name" or "Age".

🔑

Primary Key

A unique ID for each row. No two rows share it.

🔗

Foreign Key

A column that links to another table's Primary Key.

📌

This guide covers DDL, DML, Clauses and Joins

DDL

🏗️ Data Definition Language

DDL commands define the structure of your database — like creating tables, modifying columns, or deleting tables. These are like the blueprint commands.

CREATE TABLE
Create a new table
Define columns and their data types
📋 Analogy: Like creating a new sheet in Excel and naming each column header.
CREATE TABLE table_name (
  column1 datatype [constraints],
  column2 datatype [constraints]
);
create_students_table.sql 
CREATE TABLE students (
  student_id   INT           PRIMARY KEY,
  name         VARCHAR(100) NOT NULL,
  age          INT           CHECK (age >= 16),
  email        VARCHAR(200) UNIQUE,
  city         VARCHAR(50)  DEFAULT 'Chennai',
  enrolled_on  DATE
);

ALTER TABLE
Modify an existing table
Add, remove or change columns
🔧 Analogy: Like adding a new column to an existing Excel sheet, or renaming a column header.
alter_examples.sql 
-- Add a new column
ALTER TABLE students
ADD COLUMN phone VARCHAR(15);

-- Change a column's data type
ALTER TABLE students
MODIFY COLUMN age SMALLINT;

-- Rename a column
ALTER TABLE students
RENAME COLUMN city TO hometown;

-- Remove a column
ALTER TABLE students
DROP COLUMN phone;
DROP / TRUNCATE
Delete a table or clear all its data
These are destructive — use with extreme care
drop_truncate.sql 
-- DROP: deletes the table completely (structure + data)
DROP TABLE students;

-- DROP IF EXISTS: safe version (no error if table doesn't exist)
DROP TABLE IF EXISTS students;

-- TRUNCATE: deletes ALL rows but keeps the table structure
TRUNCATE TABLE students;
CommandRemoves Data?Removes Table?Can Rollback?
DROP✅ Yes✅ Yes❌ No
TRUNCATE✅ Yes❌ No❌ No
DELETE✅ Yes❌ No✅ Yes
⚠️ DROP removes the entire table forever. TRUNCATE removes all rows but keeps the table shell. Neither can be undone!

Constraints — Rules on Columns

CONSTRAINTS
Rules that protect your data quality
Applied when creating or altering a table
ConstraintWhat it meansExample
PRIMARY KEY Unique identifier for each row. Cannot be NULL. student_id INT PRIMARY KEY
NOT NULL This column must always have a value. name VARCHAR(100) NOT NULL
UNIQUE No two rows can have the same value here. email VARCHAR(200) UNIQUE
CHECK Value must pass a logical condition. age INT CHECK (age >= 16)
DEFAULT Uses this value if no value is provided. city VARCHAR(50) DEFAULT 'Chennai'
FOREIGN KEY Links to a Primary Key in another table. dept_id INT REFERENCES departments(id)
table_with_foreign_key.sql 
CREATE TABLE departments (
  dept_id   INT          PRIMARY KEY,
  dept_name VARCHAR(100) NOT NULL
);

CREATE TABLE employees (
  emp_id   INT          PRIMARY KEY,
  emp_name VARCHAR(100) NOT NULL,
  salary   DECIMAL(10,2) DEFAULT 30000.00,
  dept_id  INT,
  FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);
DML

✏️ Data Manipulation Language

DML commands interact with the actual data inside tables — reading, inserting, updating and deleting records.

SELECT
Read data from a table
The most frequently used SQL command
🔍 Analogy: Like using "Find" in Excel — you choose which rows and columns to see.
select_examples.sql 
-- Select ALL columns from a table
SELECT * FROM students;

-- Select specific columns
SELECT name, age, city
FROM students;

-- Give a column an alias (rename for display)
SELECT name AS student_name, age AS student_age
FROM students;

-- Select with calculation
SELECT name, marks, (marks * 100 / 500) AS percentage
FROM students;

INSERT INTO
Add new rows to a table
One row or multiple rows at once
📝 Analogy: Like filling in a new student registration form and saving it to the spreadsheet.
insert_examples.sql 
-- Insert one row
INSERT INTO students (student_id, name, age, city)
VALUES (1, 'Priya', 21, 'Chennai');

-- Insert multiple rows at once
INSERT INTO students (student_id, name, age, city)
VALUES
  (2, 'Arjun', 22, 'Mumbai'),
  (3, 'Meera', 20, 'Delhi'),
  (4, 'Rahul', 23, 'Bangalore'),
  (5, 'Sneha', 21, 'Chennai');
💡 The order of values must exactly match the order of columns listed in parentheses.
UPDATE
Edit existing data in a table
Always use WHERE to target specific rows
🖊️ Analogy: Like finding Priya's row in Excel and changing her city from Chennai to Hyderabad.
update_examples.sql 
-- Update one column for a specific row
UPDATE students
SET   city = 'Hyderabad'
WHERE student_id = 1;

-- Update multiple columns at once
UPDATE students
SET   city = 'Pune', age = 24
WHERE name = 'Arjun';

-- Update all rows (no WHERE — use carefully!)
UPDATE students
SET   city = 'Unknown';  -- changes EVERY row!
⚠️ Always use WHERE with UPDATE! Without WHERE, every single row in the table gets changed.
DELETE
Remove rows from a table
Always use WHERE unless clearing all rows intentionally
🗑️ Analogy: Like selecting a row in Excel and pressing Delete to remove it.
delete_examples.sql 
-- Delete one specific row
DELETE FROM students
WHERE student_id = 3;

-- Delete rows matching a condition
DELETE FROM students
WHERE city = 'Delhi';

-- Delete ALL rows (keeps the table structure)
DELETE FROM students;
⚠️ Always use WHERE with DELETE! Without WHERE, every row is deleted permanently.
Clauses

🎛️ SQL Clauses

Clauses are added to your SELECT queries to filter, sort, group, and limit results. They're the power tools that make SQL so flexible.

WHERE
Filter rows by a condition
Like clicking the "Filter" button in Excel
where_clause.sql 
-- Simple condition
SELECT * FROM students
WHERE city = 'Chennai';

-- Number comparison
SELECT * FROM students
WHERE age > 20;

-- AND: both conditions must be true
SELECT * FROM students
WHERE city = 'Chennai' AND age >= 21;

-- OR: at least one condition must be true
SELECT * FROM students
WHERE city = 'Chennai' OR city = 'Mumbai';

-- NOT: negate a condition
SELECT * FROM students
WHERE NOT city = 'Delhi';

-- IN: match a list of values
SELECT * FROM students
WHERE city IN ('Chennai', 'Mumbai', 'Delhi');

-- BETWEEN: value in a range (inclusive)
SELECT * FROM students
WHERE age BETWEEN 18 AND 25;

-- LIKE: search by partial text (% is wildcard)
SELECT * FROM students
WHERE name LIKE 'P%';     -- starts with P

SELECT * FROM students
WHERE name LIKE '%a';     -- ends with a

SELECT * FROM students
WHERE name LIKE '%ri%';   -- contains 'ri'

-- IS NULL: find rows with no value
SELECT * FROM students
WHERE email IS NULL;

ORDER BY
Sort results in a specific order
ASC = A to Z (default) · DESC = Z to A
↕️ Analogy: Like clicking a column header in Excel to sort rows alphabetically or numerically.
order_by.sql 
-- Sort by name A → Z
SELECT * FROM students
ORDER BY name ASC;

-- Sort by age highest first
SELECT * FROM students
ORDER BY age DESC;

-- Sort by city first, then by name within each city
SELECT * FROM students
ORDER BY city ASC, name ASC;
DISTINCT
Remove duplicate values
Show each unique value only once
distinct.sql 
-- Get unique cities (no duplicates)
SELECT DISTINCT city FROM students;

-- Count unique cities
SELECT COUNT(DISTINCT city) AS unique_cities
FROM students;
LIMIT / OFFSET
Control how many rows to return
Great for pagination and previewing data
limit_offset.sql 
-- Show only the first 5 rows
SELECT * FROM students LIMIT 5;

-- Skip first 5, show next 5 (page 2)
SELECT * FROM students
LIMIT 5 OFFSET 5;

-- Top 3 oldest students
SELECT name, age
FROM students
ORDER BY age DESC
LIMIT 3;
GROUP BY
Group rows and calculate summaries
Like pivot tables in Excel
📊 Analogy: "How many students are from each city?" — GROUP BY bunches rows together so you can count, sum or average per group.
group_by.sql 
-- Count students from each city
SELECT city, COUNT(*) AS total_students
FROM   students
GROUP BY city;

-- Average age per city
SELECT city, AVG(age) AS avg_age
FROM   students
GROUP BY city
ORDER BY avg_age DESC;

HAVING
Filter groups after GROUP BY
WHERE filters rows · HAVING filters groups
🎯 Analogy: "Show me only cities that have MORE than 1 student." — HAVING is WHERE applied after grouping.
having_clause.sql 
-- Cities with more than 1 student
SELECT   city, COUNT(*) AS total
FROM     students
GROUP BY city
HAVING   COUNT(*) > 1;

-- Departments where avg salary > 50000
SELECT   dept_id, AVG(salary) AS avg_salary
FROM     employees
GROUP BY dept_id
HAVING   AVG(salary) > 50000;

Aggregate Functions

COUNT · SUM · AVG · MIN · MAX
Calculate totals, averages, and extremes
Used with SELECT and GROUP BY
aggregate_functions.sql 
-- COUNT: how many rows?
SELECT COUNT(*)        AS total_students   FROM students;
SELECT COUNT(email)    AS with_email       FROM students;  -- excludes NULLs

-- SUM: total of a numeric column
SELECT SUM(salary)    AS total_payroll    FROM employees;

-- AVG: average of a numeric column
SELECT AVG(salary)    AS average_salary   FROM employees;

-- MIN and MAX
SELECT MIN(age)       AS youngest         FROM students;
SELECT MAX(age)       AS oldest           FROM students;

-- Multiple aggregates in one query
SELECT
  COUNT(*)     AS total,
  AVG(salary)  AS avg_salary,
  MIN(salary)  AS min_salary,
  MAX(salary)  AS max_salary
FROM employees;

Joins

🔗 SQL Joins

Joins combine rows from two or more tables based on a related column. When data is split across tables, joins bring it back together.

INNER JOIN

Only matching rows in both tables

LEFT JOIN

All left rows + matching right rows

RIGHT JOIN

All right rows + matching left rows

FULL OUTER JOIN

All rows from both tables

INNER JOIN
Only rows with a match in BOTH tables
Unmatched rows are excluded from results
🤝 Analogy: Two guest lists — show only people who appear on BOTH lists.
inner_join.sql 
SELECT
  s.name,
  s.city,
  c.course_name
FROM       students  AS s
INNER JOIN courses   AS c
       ON  s.student_id = c.student_id;
Result — Rahul excluded (no courses), course_id 104 excluded (no matching student)

FULL OUTER JOIN
All rows from BOTH tables
Unmatched rows from either side show as NULL
🌐 Analogy: Show ALL students and ALL courses — match them where possible, fill NULL where there's no match on either side.
full_outer_join.sql 
SELECT
  s.name,
  c.course_name
FROM            students  AS s
FULL OUTER JOIN courses   AS c
            ON  s.student_id = c.student_id;

-- MySQL doesn't support FULL OUTER JOIN directly.
-- Workaround: LEFT JOIN UNION RIGHT JOIN
SELECT s.name, c.course_name
FROM students s LEFT JOIN courses c ON s.student_id=c.student_id
UNION
SELECT s.name, c.course_name
FROM students s RIGHT JOIN courses c ON s.student_id=c.student_id;
CROSS JOIN
Every row from left × every row from right
Creates all possible combinations
🎲 Analogy: You have 3 shirt sizes and 4 colors — CROSS JOIN gives you all 12 combinations (3×4).
cross_join.sql 
-- Every student paired with every course (useful for scheduling)
SELECT
  s.name,
  c.course_name
FROM        students  s
CROSS JOIN courses   c;
-- 4 students × 4 courses = 16 rows returned
SELF JOIN
A table joined with itself
Useful for hierarchical or comparative data
🪞 Analogy: An employee table where each employee has a "manager_id" that refers back to another employee in the same table.
self_join.sql 
-- employees table has: emp_id, name, manager_id
-- manager_id references emp_id in the SAME table

SELECT
  e.name       AS employee,
  m.name       AS manager
FROM      employees  AS e
LEFT JOIN employees  AS m
      ON  e.manager_id = m.emp_id;

Putting It All Together — Full Example

🏆
A real-world query using everything you've learned
Students enrolled in courses, grouped by city, filtered by count
full_example_query.sql 
-- Find cities where more than 1 student is enrolled in a course,
-- show the city, number of enrolled students, and average age

SELECT
  s.city,
  COUNT(DISTINCT s.student_id) AS enrolled_students,
  AVG(s.age)                     AS average_age

FROM       students  AS s
INNER JOIN courses   AS c
       ON  s.student_id = c.student_id

WHERE    s.age BETWEEN 18 AND 30

GROUP BY s.city
HAVING   COUNT(DISTINCT s.student_id) > 1

ORDER BY enrolled_students DESC
LIMIT    10;
💡 This single query uses: SELECT, INNER JOIN, WHERE, BETWEEN, GROUP BY, HAVING, COUNT, AVG, DISTINCT, ORDER BY and LIMIT — all in one!
